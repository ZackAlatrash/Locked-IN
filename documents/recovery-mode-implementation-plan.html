A4 Fix DevOptions seed so recovery entry can be tested Claude Code P0 🔹 Small · 1–2 h Not Started ▾

What To Do

Add a dedicated DevOptions recovery simulation scenario (e.g. inRecovery) that correctly seeds recovery state so the popup and all recovery flows can be manually tested. The existing seeds (usedForAWhile, overloadedWeek) do not trigger recovery because they create protocols at appClock.now but run historical ticks starting 19 days in the past — since lock.startDate = today, evaluateDailyComplianceIfNeeded finds no history before the lock period, and evaluateWeekCatchUp sees no closed weeks, so zero violations accumulate. The new scenario should either: (a) create protocols with a backdated startDate matching the historical tick window so violations accumulate naturally, or (b) directly set protocol states and window violation counts to the threshold, then call runDailyIntegrityTick to let handleRecoveryTransition detect the entry and show the popup.

Root Cause

DevOptionsController.seedUsedForAWhile calls createNonNegotiable(definition:totalLockDays:referenceDate: appClock.now) which sets lock.startDate = today. The seed then runs ticks at historical dates (today − 19 … today). evaluateDailyComplianceIfNeeded computes evaluationStart = max(rawStart, lockStart) = today and evaluationEnd = yesterday_of_tick; since today > yesterday_of_tick the guard fires and no daily violations are recorded. evaluateWeekCatchUp uses the same lock.startDate as its earliest start, so no closed weeks exist to evaluate. Result: zero weeklyViolationCount, recovery threshold never crossed.

Files

LockedIn/Application/DevOptionsController.swift (seedUsedForAWhile, seedOverloadedWeek, new seedInRecovery)

Risk If Skipped

All recovery mode features are untestable via DevOptions. Any regression in recovery entry, popup flow, pause action, or clean-day exit goes undetected until it reaches a real user who has legitimately violated enough to enter recovery.

Acceptance Criteria

✓

✓
The pause-selection step (if applicable) lists only .active protocols as candidates
